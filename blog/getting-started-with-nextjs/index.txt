<p>Next.js has revolutionized the way we build React applications by providing a powerful framework that handles many complex tasks out of the box. In this comprehensive guide, we'll explore everything you need to know to get started with Next.js.</p>
<h2>What is Next.js?</h2>
<p>Next.js is a React framework that enables functionality such as server-side rendering and generating static websites for React-based web applications. It's built by Vercel and has become one of the most popular choices for building modern web applications.</p>
<h3>Key Features</h3>
<ul>
<li><strong>Server-Side Rendering (SSR)</strong>: Render pages on the server for better SEO and performance</li>
<li><strong>Static Site Generation (SSG)</strong>: Pre-render pages at build time</li>
<li><strong>API Routes</strong>: Build API endpoints within your Next.js application</li>
<li><strong>File-based Routing</strong>: Automatic routing based on file structure</li>
<li><strong>Built-in CSS Support</strong>: Support for CSS Modules, Sass, and CSS-in-JS</li>
<li><strong>Image Optimization</strong>: Automatic image optimization and lazy loading</li>
<li><strong>TypeScript Support</strong>: First-class TypeScript support</li>
</ul>
<h2>Setting Up Your First Next.js Project</h2>
<p>Let's start by creating a new Next.js project:</p>
<pre><code class="language-bash">npx create-next-app@latest my-nextjs-app
cd my-nextjs-app
npm run dev
</code></pre>
<p>This will create a new Next.js application with all the necessary dependencies and configuration files.</p>
<h3>Project Structure</h3>
<p>A typical Next.js project has the following structure:</p>
<pre><code>my-nextjs-app/
├── pages/
│   ├── api/
│   ├── _app.js
│   └── index.js
├── public/
├── styles/
├── next.config.js
└── package.json
</code></pre>
<h2>Understanding Next.js Routing</h2>
<p>Next.js uses file-based routing, which means the file structure in your <code>pages</code> directory determines your application's routes.</p>
<h3>Basic Routing</h3>
<ul>
<li><code>pages/index.js</code> → <code>/</code></li>
<li><code>pages/about.js</code> → <code>/about</code></li>
<li><code>pages/blog/index.js</code> → <code>/blog</code></li>
<li><code>pages/blog/[slug].js</code> → <code>/blog/:slug</code></li>
</ul>
<h3>Dynamic Routes</h3>
<p>Dynamic routes are created using square brackets:</p>
<pre><code class="language-javascript">// pages/blog/[slug].js
import { useRouter } from 'next/router'

export default function BlogPost() {
  const router = useRouter()
  const { slug } = router.query

  return &#x3C;div>Post: {slug}&#x3C;/div>
}
</code></pre>
<h2>Data Fetching Methods</h2>
<p>Next.js provides several methods for fetching data:</p>
<h3>getStaticProps</h3>
<p>Used for Static Site Generation (SSG):</p>
<pre><code class="language-javascript">export async function getStaticProps() {
  const res = await fetch('https://api.example.com/posts')
  const posts = await res.json()

  return {
    props: {
      posts,
    },
    revalidate: 60, // Revalidate every 60 seconds
  }
}
</code></pre>
<h3>getServerSideProps</h3>
<p>Used for Server-Side Rendering (SSR):</p>
<pre><code class="language-javascript">export async function getServerSideProps(context) {
  const res = await fetch(`https://api.example.com/posts/${context.params.id}`)
  const post = await res.json()

  return {
    props: {
      post,
    },
  }
}
</code></pre>
<h3>getStaticPaths</h3>
<p>Used with dynamic routes for SSG:</p>
<pre><code class="language-javascript">export async function getStaticPaths() {
  const res = await fetch('https://api.example.com/posts')
  const posts = await res.json()

  const paths = posts.map((post) => ({
    params: { slug: post.slug },
  }))

  return { paths, fallback: false }
}
</code></pre>
